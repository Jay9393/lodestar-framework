# 3 · Build — tasks

**Purpose:** implementation notes, one per task.

- **File naming:** `TASK-NNN-<slug>.md` · **parent:** `US-NNN`
- **Rules:** every task traces to a User Story. Commit/PR messages include
  `TASK-NNN`. If implementation must diverge from spec, **update the spec first**.
  Tests verify the US's ACs.
- **Definition of done (→ operate):** ACs met, tests pass, deployable.

Full guidelines: `../../.lodestar/SPEC.md` → §5.3 / `#build`.
