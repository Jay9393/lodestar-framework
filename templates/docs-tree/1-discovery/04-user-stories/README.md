# 04 · User Stories

**Purpose:** break PRDs into testable, behavior-level units.

- **File naming:** `US-NNN-<slug>.md` · **parent:** `PRD-NNN`
- **Split when:** user goal, action, state transition, role/permission,
  success/failure context, or testable unit differs.
- **Each story:** who-what-why + testable acceptance criteria.
- **Definition of done (→ design):** every story is a testable unit with ACs.

Full guidelines: `../../../.lodestar/SPEC.md` → §5.1 / `#s04`.
