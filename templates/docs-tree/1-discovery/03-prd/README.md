# 03 · PRD

**Purpose:** specify *problem-solving units* — not feature lists.

- **File naming:** `PRD-NNN-<slug>.md` (one problem per file) · **parent:** `PLAN-NNN`
- **Split when:** the user problem, policy, release unit, key state, or data
  lifecycle differs.
- **Required sections (8):** user problem · scope · policy · glossary ·
  screens/flows · edge cases · permissions · **acceptance criteria** (verifiable:
  subject + condition + expected result).
- **Definition of done (→ 04):** all 8 sections present, ACs verifiable.

Full guidelines: `../../../.lodestar/SPEC.md` → §5.1 / `#s03`.
